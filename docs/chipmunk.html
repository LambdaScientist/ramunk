An FFI to the Chipmunk Game Dynamics C-library.

Please refer to the official documentation for a complete overview of all procedures.

1 Introduction

The library was originally written by Jay McCarthy and later updated by Freezerburn and Sam Vervaeck.

Chipmunk Physics is an FFI to the Chipmunk physics engine, which means that it uses external binaries that interact with racket. For this reason, some platforms might not be supported. You will need to recompile the binary for your specific platform or file a request in one of the repositories.

One of the main advantages of this library is its speed. You can try out some of the demos to verify this for yourself. For this reason this library is extremely well-suited for game applications and similar programs that require smooth physical interactions.

2 FFI Bindings

This section documents all of the procedures that are available to the end-user.

 (require "chipmunk-ffi.rkt")

This module contains all of the bindings to the Chipmunk binaries. There is no need to import additional files.

2.1 Vectors

struct

(struct cpVect (x y)
  #:extra-constructor-name cpv)
  x : cpFloat?
  y : cpFloat?

The representation of a two-dimensional vector in Chipmunk.

procedure

(cpvadd v1 v2) → cpVect?

  v1 : cpVect?
  v2 : cpVect?

Produce a new cpVect that is the sum of v1 and v2.

procedure

(cpvsub v1 v2) → cpVect?

  v1 : cpVect?
  v2 : cpVect?

Produce a new cpVect by subtracting v1 from v2.

procedure

(cpvneg v) → cpVect?

  v : cpVect?

Take the additive inverse of v and return it as a new cpVect.

2.2 Shapes

A shape is used by the Chipmunk library to determine collisions between bodys. Shapes that are added to a cpSpace will be able to collide with each other.
