An experimental physics engine for the Racket language.

1 Introduction

This library provides a 2D-physics engine for the Racket language. While this is still a work in progress, you can already try out some of the demos to get a feeling of what is availiable. Currently only basic physical interations are supported, such as shape-to-shape collisions and simple constraints.

The library uses the Chipmunk Game Dynamics C-library internally, which means that some external binaries are required for this library to work. Some platforms might not be supported. If you would like to add support for a certain platform you are invited to update the source code and merge it with the main repository.

2 API Reference

This section provides an overview of all of the classes that are availiable.

2.1 Vectors

Vectors in this library are ordinary racket vectors containing only inexact numerical values.

procedure

(vector c ...) → vector?

  c : inexact?
Produces an ordinary racket vector. The elements of the vector have to be inexact? numbers.

procedure

(vector-add v ...) → vector?

  v : vector?
Produce a new vector that is the sum of all v’s.

procedure

(vector-sub v ...) → vector?

  v : vector?
Produce a new vector by subtracting all v’s.

procedure

(vector-add! v ...) → vector?

  v : vector?
Like vector-add, but mutates the first vector instead of constructing a new one.

procedure

(vector-sub! v ...) → vector?

  v : vector?
Like vector-sub, but mutates the first vector instead of constructing a new one.

2.2 Shapes

class

shape% : class?

  superclass: object%

A shape is used to determine collisions between bodys. Shapes that are added to a space% will be able to collide with each other.

2.2.1 Segment Shapes

class

segment-shape% : class?

  superclass: shape%

A primitive line shape.

constructor

(new segment-shape% 
    [elasticity elasticity] 
    [body body]) 
 → (is-a?/c segment-shape%)
  elasticity : 0.0
  body : (is-a/c? body%)
Create a new shape% with the given elasticity. The elasticity determines the resulting velocity of a body after a collision with the shape. The body is required when constructing a new shape.

method

(send a-segment-shape get-elasticity) → inexact?

Gives back the elasticity of this shape.

method

(send a-segment-shape set-elasticity elasticity) → void?

  elasticity : inexact?
Change the elasticity of this shape to a new value. The value must be an inexact number and is usually layered between 0.0 and 1.0.

method

(send a-segment-shape get-body) → (is-a/c? body%)

Gives back the body% object this shape belongs to.

method

(send a-segment-shape set-body body) → (is-a/c? body%)

  body : (is-a/c? body%)
Binds the shape to the designated body.

2.3 Bodys

class

body% : class?

  superclass: object%

Represenation of a physical body.

constructor

(new body%    
    [[mass mass]    
    [moment moment]    
    [position position]    
    [velocity velocity]]) → (is-a?/c body%)
  mass : inexact? = #f

Constructs a new body. When no mass and moment are given, the body is considered to be a static body.

method

(send a-body get-angular-velocity) → inexact?

Gives back the angular velocity of this body in radians.

3 Chipmunk Library

This section documents the procedures that are directly available using the Chipmunk game engine. While the procedures in this section might be faster than their object-oriented counterparts when correctly used, it is recommended to use the default API for better language integration and stability.

There are two flavours of of bindings availiable: the safe bindings have a racket-like naming convention while the unsafe binings have C-style naming conventions. It is highly recommended not to use the unsafe bindings unless you have a really good reason to do so. Incorrect usage of some of those procedures can easily lead to memory corruption and program crashes.

Please refer to the official documentation for a complete overview of all procedures. Some other usefull links:

  • Chipmunk API Reference — The official API reference

  • Chipmunk Game Dynamics Manual — The official manual

3.1 Safe bindings

 (require "chipmunk.rkt")

This module contains "safe" bindings to the procedures as defined in the Chipmunk binaries.

3.2 Unsafe bindings

 (require "chipmunk-ffi.rkt")

This module contains all of direct bindings to the Chipmunk binaries. Require this file if you want low-level access to the chipmunk FFI bindings.
